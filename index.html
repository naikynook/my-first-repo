Digital Object 4: Geospatial Structures

Geospatial structures locate data in place — coordinates, neighborhoods, and the uneven surfaces of a city. This section will explore how maps and spatial layouts can turn location into a visual field, asking what becomes visible when incidents are read through geography rather than only through time or relationship.

Digital Object 3: Relational Structures

NYPD publishes shootings as three related tables — incidents, victims, and offenders — joined by a shared incident key. This D3 network makes those relationships visible: offenders flow into boroughs, boroughs into victims. Node size shows volume; link thickness shows how often a path appears; redder links carry a higher share of statistical murders. Hover any node or edge for counts, and drag nodes to inspect the structure. The goal is to bring in other datasets related to inequality to contextualize the distribution within a broader context related to class divisions and disparities.

Circular Relations

The same three related shooting tables, redrawn as a chord diagram: offender race on one side of each ribbon, victim race on the other. The palette is intentionally bright so to sit against the gravity of the data. Ribbon width is volume; hover a group or chord to isolate a relationship and see murder counts inside it.

Digital Object 2: Temporal Structures

Temporal structures organize change over time, turning sequences of days and years into something you can see at once. Rather than treating data as a static table, this section asks how a calendar can become a matrix panel — a surface where time itself is the layout.

Shootings Calendar

Keeping in theme with using code as a visual tool, I used D3 to abstract the data, prioritizing visuals over readability. Below is data for NYC's shootings from 2006 - present, playfully shown as a calendar visualization/matrix panel. Working with a real dataset and seeing D3 examples of all the beautiful possibilities made me question art's relationship to data.

Digital Object 1: Spatial Canvases

I chose to use the P5.js and three.js libraries to explore how code and web design can be used, perhaps with a projector, to create visual environments. Music visuals at concerts are a big inspiration and aspiration for what I hope code and web design can do for me.

Orbiting Grid Sphere

A stationary Three.js sphere wrapped in a white cell grid. Drag to orbit the camera around it, or scroll to zoom in and out. The grid texture is drawn on a canvas and mapped onto the sphere surface, while OrbitControls move the camera instead of rotating the object itself.

Interactive Sphere Matrix

An interactive Three.js sphere that behaves like a matrix panel wrapped in 3D. Use the sliders to change grid density and background color, scroll to zoom, and click the sphere to send a rainbow wave across individual cells. The code redraws a canvas texture each frame, measuring distance from the clicked cell to color only the squares inside a moving wave band.

Static Grid Configuration

A fixed 10×10 matrix of white squares on a black field with even spacing between cells. The canvas size is calculated from the number of columns, rows, square size, and spacing, then nested loops place each cell at its computed x and y position. The sketch runs once with noLoop(), treating the grid as a precise spatial layout rather than an animation.

2D Matrix Wave

A flat p5.js grid where each square acts as an individual cell in a matrix panel. Click anywhere to trigger a radiating rainbow wave from that point. On each frame, the code expands a circular wave radius and tests every cell’s distance from the click; only cells inside the wave front receive shifting HSB colors before returning to white.
