# Portfolio Website Style Report

Reference site: [Adam Vosburgh](https://adamvosburgh.com/)

## Overall Direction

The Adam Vosburgh portfolio has a very minimal, utilitarian, and web-native style. It feels less like a polished marketing website and more like a clean project index or personal operating system. The design is quiet, direct, and intentionally stripped down. It does not rely on large hero images, decorative graphics, gradients, or heavy branding. Instead, the personality comes from restraint, spacing, plain language, and the way projects are organized as simple links.

The site should feel intelligent, lightweight, and slightly experimental. It should look like a working archive for creative technical projects rather than a conventional portfolio template.

## Visual Keywords

- Minimal
- Index-like
- Functional
- Sparse
- Quiet
- Technical
- Archive-based
- Web-native
- Unbranded
- Slightly mysterious

## Layout Style

The layout should be simple and open. Content should sit on a plain background with generous empty space around it. The page should avoid boxed sections, cards, shadows, rounded panels, and colorful blocks. A visitor should immediately understand that the site is primarily a collection of projects, not a decorative landing page.

The top of the page should prioritize the person's name or site identity. Navigation can be presented as plain text links, with options such as list view, gallery view, apps, projects, updates, teaching, or about. These links should feel like interface controls rather than styled buttons.

Project entries should be direct and compact. They can appear as simple text links in a list, or as a very restrained gallery. If a project is private or unfinished, small notes like "password protected" can appear beside it. This adds to the feeling that the portfolio is an active archive rather than a static showcase.

## Typography

Typography should be understated and clear. A neutral sans-serif font, system font, or simple monospace-adjacent typeface would fit the reference well. The type should not feel overly branded or expressive. It should feel practical.

Recommended qualities:

- Small to medium text sizes
- Regular font weights
- Minimal use of bold
- Clear hierarchy through spacing rather than dramatic size changes
- Plain link styling
- No overly decorative headings

Headings should be direct and modest. The name at the top can be the largest element, but it should still feel quiet. Section labels and navigation items can remain close in size to body text.

## Color Palette

The color palette should be extremely restrained. The reference site appears to rely mainly on black text, white or near-white background, and subtle neutral tones. Color should be used sparingly, if at all.

Suggested palette:

- Background: white or very light gray
- Primary text: black or near-black
- Secondary text: gray
- Links: black, dark gray, or default blue if a raw-web feel is desired
- Borders: very light gray, used only when necessary

Avoid bright gradients, saturated backgrounds, drop shadows, and decorative color systems. The site should feel clean and almost unfinished in a deliberate way.

## Navigation And Interaction

Navigation should feel immediate and lightweight. Text links can sit close together at the top of the page, almost like filters or tabs. The reference includes ideas like "List View," "Gallery View," and "Apps," which makes the portfolio feel interactive without needing heavy UI design.

Interactions should be subtle:

- Links can underline on hover
- View toggles can switch between list and gallery layouts
- Project previews can stay minimal
- Protected or unavailable projects can be labeled plainly
- A small close symbol or simple overlay can be used if needed

Buttons should be avoided unless there is a strong reason to use them. Most actions can be represented as text.

## Image And Media Treatment

Images should not dominate the page unless the user chooses gallery view. In list view, the site can work without images entirely. In gallery view, images should be arranged simply, with little decoration. The images should feel like documentation of projects rather than polished promotional thumbnails.

If using images, keep them:

- Flat
- Unframed or lightly framed
- Consistently sized
- Clearly connected to project links
- Secondary to the project archive structure

## Content Voice

The writing should be concise and factual. Project names, app names, dates, and short descriptors should do most of the work. The site should not over-explain itself.

The tone should feel like:

- "Here is the work."
- "Here are the tools and projects."
- "Some things are public, some are private."
- "This is an active creative archive."

Avoid marketing language such as "innovative solutions," "cutting-edge design," or "beautiful user experiences." The style is stronger when it stays plain and specific.

## Applying This Style To My Website

For my portfolio website, I should move toward a calmer and more minimal visual system. The current page can become more like an archive by reducing decorative styling and focusing on hierarchy, structure, and project links.

Recommended changes:

- Replace colorful backgrounds with a white or near-white background.
- Use black and gray text as the main visual language.
- Make the header simpler and more text-based.
- Treat navigation as small plain links instead of prominent UI elements.
- Reduce large visual effects such as gradients, shadows, and rounded containers.
- Organize work as a list of projects, sections, or apps.
- Use spacing and alignment to create hierarchy.
- Let the website feel like a living index of work rather than a finished poster.

The goal is not to copy Adam Vosburgh's website exactly, but to capture its attitude: clear, minimal, direct, archival, and quietly experimental.
