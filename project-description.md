# Portfolio Website Project Description

Reference site: [Adam Vosburgh](https://adamvosburgh.com/)

## Project Overview

This website will be a personal portfolio that presents a collection of projects in a clear, minimal, and organized way. The site should closely follow the layout logic and visual attitude of Adam Vosburgh's portfolio website, especially its simple text-based structure, direct navigation, and archive-like presentation of work.

The goal is to create a portfolio that feels like a living index of creative, academic, and professional projects. Instead of using a highly decorative design, the website should focus on clarity, structure, and easy browsing. Visitors should be able to quickly understand who the portfolio belongs to, what kinds of work are included, and how to move between different project categories.

## Main Purpose

The purpose of the website is to display various projects in one organized digital space. It should function as both a portfolio and an archive. The site should make it easy to browse through work, compare different types of projects, and access more detailed information about each one.

The portfolio should communicate that the work spans multiple contexts. Some projects may come from academic research, school studios, or coursework. Others may come from professional experience, collaborations, internships, or independent work. The website should create a simple structure that allows these different categories to coexist without feeling visually cluttered.

## Design Inspiration

The main reference is Adam Vosburgh's website. That site uses a very restrained layout with a name at the top, simple navigation links, view options, and a list of project or app links. It feels direct, minimal, and slightly experimental. The design does not try to over-explain the work or decorate the interface. Instead, it lets the project index become the main experience.

This portfolio should borrow that same quiet and organized feeling. It should be text-forward, easy to scan, and intentionally simple. The design should feel more like a curated archive than a traditional portfolio homepage with large graphics, animations, or promotional language.

## Key Layout Requirements

The website should include a clear header area with the portfolio owner's name or identity. The name should be one of the first things visible on the page, similar to the way Adam Vosburgh's site immediately establishes ownership.

Below or near the header, the site should include view options such as list view and gallery view. These controls would allow visitors to browse the same project archive in different ways. List view should emphasize project titles and categories, while gallery view can introduce images or thumbnails if needed.

The main section of the page should contain project category tabs. These tabs should help organize the work and replace some of the categories found on the reference site.

Required main section tabs:

- All
- Academic
- Professional

The "All" tab should remain because it gives visitors a complete view of every project. The "Academic" tab should filter or organize projects related to school, research, design studios, coursework, experiments, and computational design studies. The "Professional" tab should highlight work connected to jobs, internships, clients, collaborations, or practice-based experience.

## Navigation Differences From The Reference

The website should closely follow the reference layout, but with several important changes. The reference website includes navigation items such as "Teaching," "Updates," and "Apps." This portfolio should not include those exact sections.

Changes to make:

- Do not include a "Teaching" tab.
- Do not include an "Updates" tab.
- Do not place an "Apps" tab in the far corner.
- Keep the "All" category.
- Add "Academic" and "Professional" as main section tabs.
- Keep the "About" tab positioned at the far right.

The "About" tab should remain separate from the main project filtering tabs. It should sit at the far right of the navigation area, acting as a clear place for visitors to learn more about the portfolio owner, their background, interests, and contact information.

## Project Organization

Projects should be presented as a simple archive. Each project entry could include a title, year, category, and short descriptor. The design should avoid making every project look like a large card unless the user is in gallery view. In list view, the projects can appear as clean text links, keeping the interface compact and easy to scan.

Each project can belong to one or more categories:

- Academic projects
- Professional projects
- Personal or experimental projects, if included in the "All" view

The "All" tab should show every project together. The "Academic" tab should narrow the archive to school-related and research-based work. The "Professional" tab should narrow the archive to practice-based work.

## User Experience

The website should feel fast, lightweight, and direct. A visitor should not have to scroll through a long introduction before seeing the work. The project archive should be the main focus of the page.

The experience should support quick browsing:

- Visitors can immediately see the portfolio identity.
- Visitors can choose between viewing all projects or filtered categories.
- Visitors can switch between list and gallery views.
- Visitors can access the about section from the far right.
- Visitors can click project names to learn more.

The overall interaction should feel simple and intentional. The site should not rely on complicated animations or large transitions. Any interactive elements should support browsing and organization.

## Content Tone

The writing on the website should be concise, clear, and confident. It should describe projects plainly rather than using overly promotional language. The tone should match the minimal layout: direct, organized, and professional.

Project descriptions should explain what the project is, what tools or methods were used, and why the work matters. The language can be thoughtful, but it should stay focused and readable.

## Final Direction

This portfolio website should become a clean digital archive of work. It should use Adam Vosburgh's website as a strong reference for structure, simplicity, and tone, while adjusting the navigation to better fit the intended project categories.

The final site should include the feeling of a personal index: minimal, organized, easy to browse, and centered on the projects themselves. The most important changes from the reference are the removal of "Teaching," "Updates," and the corner "Apps" tab, and the addition of clear project tabs for "All," "Academic," and "Professional," with "About" remaining at the far right.
